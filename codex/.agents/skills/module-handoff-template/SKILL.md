# 模块交接

## 适用时机

- 模块实现、接口接入或收口检查已经完成
- 需要把实现范围、验证方式和剩余风险整理给后续维护者或联调方
- 需要形成一个可追踪的交接文档

## 目标与边界

- 目标是生成交接文档，汇总已经完成的实现、已知限制和验证方式
- 不补写业务代码，不替产品或后端确认仍缺失的信息

## 输出文件

生成 `<handoff_doc_path-docs/modules/<feature>-handoff.md>`

## 必须包含

1. 路由路径
2. `<project_pages_dir-src/pages/>` 下的主要文件
3. `<project_service_dir-src/service/>` 新增或修改的 API 列表
4. 已知限制
5. 本地验证方式，例如 `<dev_command-yarn dev>`、`<lint_command-yarn lint>`
6. 联调联系人表头，可留空待人工补

## 硬规则

- 不要泄露密钥。
- 不要粘贴 `.env`。
- 如果 `<feature>` 文件名不明确，先根据需求卡文件名或用户描述确认。

## 收尾建议

- 结束时说明交接文档路径。
- 如果仍有产品、设计、接口或联调联系人缺口，明确列为待人工补充项。
