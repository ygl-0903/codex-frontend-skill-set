# 模块收口

## 适用时机

- 页面骨架、真实数据或主要交互已经基本完成
- 需要在交付前检查文案、状态、可访问性和同域页面一致性
- 需要确认是否仍有产品、设计或接口契约风险

## 目标与边界

- 目标是收口体验质量和交付风险，不重新拆需求、不重搭页面结构
- 只处理明显缺口和一致性问题；大范围重构应单独拆出后续任务

## 检查重点

1. 对照需求卡中的状态机，检查 loading / error / empty / success。
2. 中文用户文案与现有页面语气保持一致。
3. 检查按钮、焦点、基础可访问性是否存在明显缺口。
4. 对照附近页面，确认布局、间距、层级、交互没有明显跑偏。

## 验证

- 跑 `<lint_command-yarn lint>`
- 必要时跑 `<build_command-yarn build>`

## 输出

- 列出仍需产品或设计确认的点。
- 如果没有待确认项，明确说明当前剩余风险主要来自什么。
