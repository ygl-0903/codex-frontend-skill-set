# 参考页复用开发通用模版

## 适用时机

- 目标页面和仓库里某个旧页面在内容结构、交互骨架和主要功能上大体相同。
- 目标页面复用同一套接口链路，请求方式、参数结构和返回结构不变。
- 本轮只需要改少量字段取值、文案、页面标题、路由或局部展示区块。
- 希望直接基于参考页完成页面和功能开发，不再拆成 `module-scaffold`、`module-api` 两轮。

## 不适用时机

- 新页面引入了新流程、新状态、新弹窗链路或新的提交动作。
- 接口虽然看起来类似，但请求参数、返回字段、成功判断或调用时序已经变化。
- 参考页和目标页只是视觉接近，业务语义、状态流或交互目标并不一致。
- 当前需求还没有明确参考页，或者参考页本身就存在明显历史问题，不能继续复制。

## 先读什么

1. 如果有 `docs/modules/<feature>.md`，先读它；没有就读用户需求、页面注册入口和目标页面入口说明。
2. 读取 `AGENTS.md` 与 `bd-fe-conventions`。
3. 读取参考页本身，以及必要的同目录组件、样式文件和关联子页。
4. 读取 `<project_request_entry-@/utils/request.js>`、`<project_auth_entry-@/utils/auth.js>`、`<api_contract_source-config/api.js>`，确认接口与登录态链路确实可直接复用。

## 执行步骤

1. 先锁定至少 1 个主参考页；如果存在详情页、结果页、弹窗组件或子列表联动，再补 1 个最关键的关联参考文件。
2. 先判断是否满足“同构复用”：
   - 页面主体区块基本一致。
   - 接口链路一致。
   - 只需要轻量修改字段、文案、标题、路由。
   - 如果以上任一项不成立，停止使用本 skill，退回正常模块流程。
3. 基于参考页直接复用这些内容：
   - 页面目录结构
   - 页面主体区块组织
   - 主要状态、事件处理和提交逻辑
   - 接口调用方式
   - loading / empty / error / success 状态处理
   - 关联组件或局部样式的复用方式
4. 只改必要内容：
   - 新页面路径和 `<project_route_registration_pattern-pages.json>` 路由注册
   - 页面标题、导航文案、按钮文案、提示语
   - 少量展示字段取值
   - 与目标页面直接相关的跳转路径
5. 改动时保持原有接口语义和状态流，不要为了“顺手统一”重写接口层、重组数据结构或新增抽象。
6. 完成后自查页面渲染、接口联动、跳转、空态、报错提示和目标运行端交互是否仍然成立。

## 硬规则

- 必须先给出参考页路径，再开始复用开发。
- 请求统一走 `<project_request_entry-@/utils/request.js>`，不要新增第二套封装。
- 涉及登录态时优先复用 `<project_auth_entry-@/utils/auth.js>`，不要自己补一套缓存逻辑。
- 没有真实接口依据时，不要借复用之名猜字段。
- 只允许轻量改字段消费；如果字段语义已经变了，就不再属于本 skill。
- 优先直接复用参考页已有功能，不要把本来可复用的行为重新写一遍。
- 如果复制后发现需要连续修改多个方法、多个状态对象或整段请求解析，说明已经超出“少量改动”范围，应停止并切回常规模块开发。

## 标准触发模板

适合直接给 AI 的请求模板：

`使用 module-reference-reuse-template，参考 [参考页路径]，开发 [目标页路径]。目标页面与参考页内容大体相同，接口保持一致，只改 [字段/文案/路由]。请直接复用参考页的页面结构、接口调用和功能，不走 mock 和分阶段开发。`

推荐补齐这些输入：

- 参考页路径
- 目标页路径
- 需要修改的字段名、文案或跳转路径
- 是否需要同步复用关联组件或子页
- 是否已有 `docs/modules/<feature>.md`

## 输出要求

- 优先列出参考页路径、目标页路径和关联组件路径，不贴大段源码。
- 明确说明这轮是否满足：
  - 同构复用，可直接开发
  - 部分接近，但已超出复用边界
- 如果超出边界，要直接指出是接口变了、流程变了，还是页面结构已经不是同一类。

## 收尾

- 至少确认新页面已按 `<project_route_registration_pattern-pages.json>` 正确注册，并与对应路由域、包归属或导航入口一致。
- 至少确认复用过来的接口、跳转和状态提示没有因路径或文案调整而失效。
- 如果页面和功能已完成，下一轮通常进入 `module-polish`；如果还要补交接资料，再进入 `module-handoff`。
