# 页面局部功能删除

## 适用时机

- 需求已明确为“下线某个页面里的一个局部功能”。
- 你能提供准确的页面路径。
- 你能提供明确的 UI 定位信息，优先是 `.className`，也可以补充按钮文案、组件名或附近文案。
- 目标功能主要收敛在当前页面及该页面直接引用的本地组件里。

## 不适用时机

- 需要删除的是整页路由、分包、tab、全局组件或公共工具能力。
- 逻辑明显跨到公共工具、公共配置、共享 API 封装或多个页面共享模块。
- 同一个方法、状态或组件同时服务多个功能，无法安全判断是否只属于该 UI。
- 你给的容器标识不唯一，或页面里存在多个同名 class 无法确定目标。

## 输入要求

至少提供：

- 页面路径
- 待删除容器标识，优先 `.className`

建议补充：

- 该容器里的按钮文案 / 标题文案
- 如果 class 不够唯一，说明“以哪一段 UI 为准”

## 删除边界

只允许删除：

- 当前页面模板中对应的节点
- 当前页面状态、派生逻辑、监听、方法、生命周期里仅为该 UI 服务的逻辑
- 当前页面直接引用、且仅被该功能使用的本地组件引用逻辑
- 当前页面的直接父页面中，仅为该 UI 透传、组装或派生的字段 / 对象片段
- 当前页面局部样式或同目录样式文件里的对应选择器
- 删除该 UI 后，为保证当前页面视觉和布局合理所必需的高度、间距、对齐、留白调整

明确不删除：

- 路由注册文件，例如 `<project_router_file-src/router/index.tsx>` 或等价入口
- 公共工具目录
- 公共配置
- 公共 API 封装
- 其他页面复用逻辑
- 无法确认是否共享的组件、方法、状态

## 先读什么

1. `AGENTS.md`
2. `bd-fe-conventions`
3. 目标页面文件
4. 目标页面同目录样式文件
5. 目标页面直接引用、且与该功能相关的本地组件

## 执行步骤

1. 先在目标页面中定位容器节点，确认它就是用户要删除的 UI。
2. 反查该容器的：
   - 事件绑定
   - 条件渲染 / 列表渲染 / 动态 class / disabled 等绑定
   - 文案、图片、局部组件、插槽或条件渲染依赖
3. 继续追踪到当前页面内直接相关的：
   - 状态
   - 派生逻辑
   - 监听
   - 方法
   - 生命周期中的分支逻辑
   - import 的本地组件
4. 只删除“仅为该 UI 服务”的代码，不顺手重构其他逻辑。
5. 如果目标 UI 是当前页面被父页面引用的本地组件，并且父页面里存在只为该 UI 透传、组装或派生的字段：
   - 可以继续回溯到直接父页面，一并删除这部分上游组装
   - 只允许到“直接父页面”这一层，不继续跨到公共模块或更远调用链
6. 如果该 UI 使用了当前页面直接引用的本地组件，先判断该组件是否只服务这个功能：
   - 仅在该页面该功能中使用，可以一起删掉引用和相关逻辑
   - 只要存在复用或用途不明，就不要删组件本体
7. 删除完成后回查：
   - 删除后是否出现明显空白、塌陷、按钮贴边、卡片高度不合理、滚动区域异常
   - 根据当前页面现有视觉语言，顺手调整必要的高度、间距、对齐
   - 未使用 import
   - 未使用方法 / 派生逻辑 / 状态
   - 残留 class 选择器
   - 残留跳转、Toast、弹窗开关、局部文案
   - 父页面残留的透传字段、局部结果组装、派生属性
8. 输出时说明删了哪些页面内逻辑，以及哪些共享逻辑因为边界限制未动。

## 停止条件

遇到以下情况不要直接删，先向用户说明冲突点：

- `.className` 不唯一
- 同一方法同时服务多个 UI
- 同一状态被多个区块共享
- 同一本地组件在页面内被多处复用
- 父页面中的同一结果对象或派生字段同时服务多个区块
- 想删干净必须改公共模块

## 硬规则

- 先删当前页面，再决定是否删当前页面直接引用的本地组件逻辑。
- 如果当前页面是被父页面直接引用的本地组件，可以继续清理“直接父页面”里只为该 UI 服务的透传或结果组装；不要再往上游扩散。
- 不能因为“看起来没用了”就跨文件清理公共层。
- 不要改接口参数、返回解析、登录态链路，除非它们只存在于当前页面且只服务该 UI。
- 删除局部 UI 后，如果页面布局明显失衡，必须在同次修改里把高度、间距、对齐收回来，不能只删节点就结束。
- 删除后优先保证页面还能正常渲染，避免留下编译错误和明显死引用。
- 优先给出文件路径和影响范围，不贴大段源码。

## 标准触发模板

`使用 module-remove-ui-feature-template，处理 [页面路径]，删除 [.className] 对应功能。只删当前页面里的模板/脚本/样式，以及当前页面直接引用的本地组件逻辑，不要影响其他业务。`

如果 class 不够唯一，补一句：

`以“[按钮文案 / 标题文案 / 邻近区块说明]”这一块为准。`

## 输出要求

- 先说明是否已准确定位目标 UI。
- 明确列出本次实际删除的文件和范围。
- 如果有未删除的共享逻辑，明确说明原因。
- 如果无法安全删除，先列冲突点，再等用户决定。
