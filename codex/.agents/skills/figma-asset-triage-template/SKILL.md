# Figma 资产分流

## 适用时机

- 用户提供 Figma 链接、节点、截图或标注图，让你还原页面。
- 任务已经进入视觉回填，需要判断哪些元素继续用 CSS，哪些不该硬写。
- 需要把设计稿中的图片资产实际导出到用户指定目录；若未指定，默认导出到 `<project_asset_staging_dir-static/figma/>`，并登记到 `<asset_inventory_doc-docs/modules/Figma-images.md>`。
- 页面主要工作是视觉还原时，通常与现有的 UI 还原类 skill 配合使用。

## 输入补充约定

- 允许用户额外指定“资源目录”：
  - 本地目录使用用户指定的项目资源目录作为导图落点，例如 `<project_asset_staging_dir-static/figma/>` 这一类目录。
  - 若未指定，默认使用 `<project_asset_staging_dir-static/figma/>`。
- 导图清单除本地路径外，还要给出项目默认服务器图片路径：
  - 格式使用项目现有静态资源前缀 + 资源目录 + 文件名，例如 `<project_public_asset_prefix-${APP_PUBLIC_PATH}images/>[资源目录]/文件名.png`
  - 若用户未指定资源目录，则按默认目录语义处理。
- 如果用户没有特别说明页面要先接本地资源，默认认为后续页面引用应使用服务器图片路径；本地导出目录只作为上传前的中转目录。
- 如果用户明确说明“本轮不落本地，只输出导图清单和服务器路径约定”，允许跳过本地导出，但仍要按上述规则生成清单。

## 先读什么

1. `AGENTS.md`
2. `bd-fe-conventions`
3. 如果有 `docs/modules/<feature>.md`，先读需求卡；没有就读用户需求、目标页面和参考页。
4. `<asset_inventory_doc-docs/modules/Figma-images.md>`，确认已有登记格式和历史命名。

## 核心目标

- 先做元素分流：哪些应继续用 CSS 还原，哪些应导图。
- 不把适合代码表达的元素误切成图片，也不把复杂视觉硬塞给 CSS。
- 能直接拿到素材时，优先实际导出到本地目录；若用户未指定则落到 `<project_asset_staging_dir-static/figma/>` 这类项目默认中转目录。
- 拿不到素材时，明确缺失项、建议命名和用途说明。
- 最终按“页面分组 + 组内逐条图片”追加 `<asset_inventory_doc-docs/modules/Figma-images.md>`。
- 让输出清单能直接给页面还原阶段使用，做到一边改 UI、一边按清单中的服务器图片地址接图。

## 判断规则

### 优先继续用 CSS / 组件还原

- 纯色背景、线性渐变、圆角、阴影、描边、分割线。
- 常规矩形、圆形、角标、标签、胶囊按钮、进度条。
- 纯文本、数字、图标字体、项目现有图标系统可覆盖的图标，例如 `<icon_system-uni-icons>`。
- 常规卡片排版、吸底栏、列表、弹窗、头像框、徽标底板。
- 可以由 1 到 2 层容器 + 常规样式稳定实现的装饰。

### 优先导出图片

- 不规则插画、人物、吉祥物、复杂纹理、噪点、光斑、照片。
- 多层混合、复杂蒙版、路径裁切、布尔形状、位图叠加。
- 难以稳定复现的异形按钮、异形底图、复杂高光、特殊描边。
- 需要像素级还原的品牌图形、活动主视觉、徽章、证书底纹。
- 复杂图表背景、棋盘底纹、装饰性大面积纹样。

### 需要谨慎判断

- 小图标如果仓库已有现成资源或项目图标系统可替代，优先复用，不急着导图。
- 轻微装饰如果用 CSS 只要少量代码即可稳定实现，优先 CSS，不要为了省几行样式切一张图。
- 如果一个区块要靠大量绝对定位、多个伪元素、复杂渐变叠加才能勉强接近设计稿，通常应改判为导图。
- 如果一个视觉区块同时包含“插画/底图”和“业务相关文字、按钮、标签、数值”，优先导出无文字的底图或插画层，让文字、按钮和标签继续由页面代码实现。
- 只有当文字本身属于不可拆分的品牌字效、艺术标题、主视觉排版，或者 Figma 中拿不到可复用的无文案底图时，才导整卡。
- 若设计图本身分辨率不足、截图模糊或节点信息不完整，不要臆测细节；明确标注为待补素材。

## 执行步骤

1. 先锁定目标页面和输入来源：
   - Figma 链接 / 节点
   - 截图 / 标注图
   - 页面路径或模块名
   - 用户是否指定资源目录；若未指定，按 `<project_asset_staging_dir-static/figma/>` 和项目默认服务器资源目录处理
2. 通读页面结构后，把视觉元素分成三类：
   - `CSS 可还原`
   - `建议导图`
   - `信息不足，待补素材`
3. 对 `建议导图` 的元素逐条记录：
   - 所属页面
   - 元素名称
   - 用途
   - 建议文件名
   - 是否去文案导出
   - 本地建议目录
   - 默认服务器图片路径
   - 是否已拿到素材
4. 如果当前能从 Figma 链接、节点或可下载资源中拿到素材：
   - 优先实际导出图片文件，不要只停留在清单
   - 若用户指定资源目录，导出到该目录；未指定则导出到 `<project_asset_staging_dir-static/figma/>`
   - 文件名使用页面前缀 + 元素语义，例如 `growth-header-glow.png`
   - 对含业务文案的卡片，优先尝试导出“无文字底图 / 插画层”；只有无法合理拆分时才保留整卡导图
   - 避免 `1.png`、`图层-12.png` 这类无语义命名
5. 如果当前只能看到截图、标注图，或拿不到可导出的原始素材：
   - 不伪造图片
   - 在输出里明确缺少哪张图、建议命名、推荐格式、用途和默认服务器图片路径
   - 状态标记为 `待导出` 或 `待补素材`，不要误写成已落库
   - 继续推进 CSS 可还原部分
6. 更新 `<asset_inventory_doc-docs/modules/Figma-images.md>`：
   - 按页面追加一个分组
   - 分组内逐条列图片，至少包含元素名称、建议文件名、本地目录、默认服务器图片路径、用途、状态
   - 如果该页面本轮没有新增图片，也要写明“本页元素均继续使用 CSS 还原，无新增导图”

## 路径与命名

- 默认本地目录：`<project_asset_staging_dir-static/figma/>`
- 如用户指定资源目录，本地目录改为用户指定目录；不要额外再套一层默认子目录，也不要擅自改到别的目录。
- 默认服务器图片目录：`<project_public_asset_prefix-${APP_PUBLIC_PATH}images/>figma/`
- 如用户指定资源目录，默认服务器图片目录改为同一前缀下的对应资源目录。
- 文件名使用英文小写加连字符，带页面语义前缀。
- 对去文案导出的资源，文件名优先体现 `bg / illustration / deco / artwork` 等语义，不要误命名成整卡含文案资源。
- 同一页面的同类资源尽量统一前缀。
- 若页面最终要接服务器图片，优先在清单里提供完整拼接结果，供页面代码直接引用。
- 除非用户明确要求先接本地路径做临时联调，否则默认推荐页面直接使用清单中的服务器图片路径。

## 输出要求

- 优先给出：
  - 目标页面路径
  - `CSS 可还原` 清单
  - `建议导图` 清单
  - 哪些资源按“去文案导出”，哪些资源必须整卡导出
  - 已实际导出的本地资源完整路径
  - 默认服务器图片路径清单
  - 未拿到素材的缺失项
- 少贴大段代码，重点说明判断结果和落库结果。
- 如与 UI 还原类 skill 一起使用，要明确哪些视觉点已通过 CSS 处理，哪些依赖导图资源。
- 若用户后续还要接 UI，清单中的每张图都要能直接映射到页面里的一个视觉区块，避免只给文件名不给用途。

## 禁止事项

- 不要把可稳定用 CSS 实现的简单元素一律切图。
- 不要在没有素材的情况下伪造位图文件。
- 不要在已经具备 Figma 导出条件时，只给清单不导图。
- 不要在用户已经指定资源目录后，擅自改用别的目录。
- 不要把图片资源散落到页面目录或新建随意目录；未指定时统一放项目默认资源中转目录。
- 不要只导图不登记；只要本轮判断过，就同步更新 `<asset_inventory_doc-docs/modules/Figma-images.md>`。
- 不要把带业务文案、按钮、标签的整卡一股脑导出成一张图，除非已经明确判断无法拆分。
- 不要根据模糊截图臆测品牌图形、插画细节或透明边界。

## 标准触发模板

`使用 figma-asset-triage-template，基于 [Figma 链接/节点/截图] 处理 [页面路径]。资源目录为 [目录名，可选]。请判断哪些元素继续用 CSS 还原，哪些元素应导图；若一个视觉区块同时包含插画和业务文案/按钮/标签，优先导出无文案底图或插画层，不要默认整卡导出。若能从 Figma 导出，请实际导出到本地目录。若未指定资源目录，默认按 <project_asset_staging_dir-static/figma/> 和项目默认服务器图片前缀处理。请把结果追加到 <asset_inventory_doc-docs/modules/Figma-images.md>；若素材不足，请明确列出缺失项。`
