# Figma 资产分流

## 适用时机

- 用户提供 Figma 链接、节点、截图或标注图，让你还原页面。
- 任务已进入视觉回填，需要判断哪些元素继续用 CSS，哪些应导图。
- 需要把设计稿图片实际导出到本地目录，并登记到 `<asset_registry_doc_path-docs/modules/Figma-images.md>`。
- 页面主要工作是视觉还原时，通常与 `module-handUI` 配合使用。

## 输入补充约定

- 用户可指定资源目录或资源子目录；未指定时默认使用项目的图片落库目录，例如 `<project_asset_dir-src/assets/images/figma/>`。
- 页面默认使用本地导入路径；未指定更具体约定时，可按项目默认导入基址记录，例如 `<project_asset_import_base-@/assets/images/figma/>`。

## 先读什么

1. `AGENTS.md`
2. `bd-fe-conventions`
3. 如果有 `docs/modules/<feature>.md`，先读需求卡；没有就读用户需求、目标页面和参考页。
4. `<asset_registry_doc_path-docs/modules/Figma-images.md>`，确认已有登记格式和历史命名。

## 核心目标

- 先做元素分流：哪些应继续用 CSS 还原，哪些应导图。
- 不把适合代码表达的元素误切成图片，也不把复杂视觉硬塞给 CSS。
- 能导就实际导出；不能导就明确缺失项、命名和用途。
- 最终把结果按页面追加到 `<asset_registry_doc_path-docs/modules/Figma-images.md>`，让 `module-handUI` 可直接接图。

## 判断规则

### 导图格式与去重约定

- 下载或导出图片时，默认优先选择 `png` 格式；除非用户明确指定其他格式，或该资源因透明度、压缩率、尺寸策略等原因必须使用别的格式。
- 页面中循环使用、来源接近或肉眼看上去基本是同一张的图片，只导一张并记录为复用。
- 即使有轻微尺寸、裁切、阴影、描边或摆放差异，只要肉眼看不出明显不同，仍按同一素材处理。
- 只有主体内容、配色、文案、角标、装饰细节、裁切结果或视觉语义已明显不同，才按独立资源处理。

### 优先继续用 CSS / 组件还原

- 纯色背景、线性渐变、圆角、阴影、描边、分割线。
- 常规矩形、圆形、角标、标签、胶囊按钮、进度条。
- 纯文本、数字、图标字体、项目内现成图标体系或 `<ui_library-Element UI>` 可覆盖的图标。
- 常规卡片排版、吸底栏、列表、弹窗、头像框、徽标底板。
- 可以由 1 到 2 层容器 + 常规样式稳定实现的装饰。

### 优先导出图片

- 不规则插画、人物、吉祥物、复杂纹理、噪点、光斑、照片。
- 多层混合、复杂蒙版、路径裁切、布尔形状、位图叠加。
- 难以稳定复现的异形按钮、异形底图、复杂高光、特殊描边。
- 需要像素级还原的品牌图形、活动主视觉、徽章、证书底纹。
- 复杂图表背景、棋盘底纹、装饰性大面积纹样。

### 需要谨慎判断

- 小图标如果仓库已有现成资源或项目现有 UI / 图标库可替代，优先复用，不急着导图。
- 轻微装饰如果用 CSS 只要少量代码即可稳定实现，优先 CSS，不要为了省几行样式切一张图。
- 如果一个区块要靠大量绝对定位、多个伪元素、复杂渐变叠加才能勉强接近设计稿，通常应改判为导图。
- 如果一个视觉区块同时包含“插画/底图”和“业务相关文字、按钮、标签、数值”，优先导出无文字的底图或插画层，让文字、按钮和标签继续由页面代码实现。
- 只有当文字本身属于不可拆分的品牌字效、艺术标题、主视觉排版，或者 Figma 中拿不到可复用的无文案底图时，才导整卡。
- 若设计图本身分辨率不足、截图模糊或节点信息不完整，不要臆测细节；明确标注为待补素材。

## 执行步骤

1. 先锁定目标页面和输入来源：
   - Figma 链接 / 节点
   - 截图 / 标注图
   - 页面路径或模块名
   - 用户是否指定资源目录；未指定则用项目默认图片目录，例如 `<project_asset_dir-src/assets/images/figma/>`
2. 通读页面结构后，把视觉元素分成三类：
   - `CSS 可还原`
   - `建议导图`
   - `信息不足，待补素材`
3. 对 `建议导图` 的元素逐条记录：
   - 所属页面
   - 元素名称
   - 用途
   - 建议文件名
   - 是否去文案导出
   - 本地建议目录
   - 本地导入路径
   - 是否已拿到素材
4. 如果当前能从 Figma 链接、节点或可下载资源中拿到素材：
   - 优先实际导出图片文件，不要只停留在清单
   - 导出到项目约定的图片目录；未指定时优先使用 `<project_asset_dir-src/assets/images/figma/>`
   - 默认导出 `png`
   - 先判断是否重复；只要肉眼看上去基本是同一张，就复用既有文件，不重复导出
   - 文件名使用页面前缀 + 元素语义，例如 `growth-header-glow.png`
   - 对含业务文案的卡片，优先尝试导出“无文字底图 / 插画层”；只有无法合理拆分时才保留整卡导图
   - 避免 `1.png`、`图层 12.png` 这类无语义命名
5. 如果当前只能看到截图、标注图，或拿不到可导出的原始素材：
   - 不伪造图片
   - 在输出里明确缺少哪张图、建议命名、推荐格式、用途和本地导入路径
   - 状态标记为 `待导出` 或 `待补素材`，不要误写成已落库
   - 继续推进 CSS 可还原部分
6. 更新 `<asset_registry_doc_path-docs/modules/Figma-images.md>`：
   - 按页面追加分组
   - 逐条记录元素名称、建议文件名、本地目录、本地导入路径、用途、状态
   - 若本页无新增图片，写明“本页元素均继续使用 CSS 还原，无新增导图”

## 路径与命名

- 默认本地目录使用项目约定的图片根目录或默认分组目录，例如 `<project_asset_dir-src/assets/images/figma/>`
- 如用户指定资源目录，优先落到该目录对应的位置；不要额外再套一层固定子目录，也不要擅自改到别的目录。
- 默认导入路径使用项目约定的导入基址，例如 `<project_asset_import_base-@/assets/images/figma/> + 文件名`
- 如用户指定资源目录，导入路径同步切换到对应目录；不要保留旧目录前缀。
- 文件名使用英文小写加连字符，带页面语义前缀。
- 重复复用的同一素材保持同一文件名和路径，不因位置不同或循环渲染额外生成多份文件。
- 对去文案导出的资源，文件名优先体现“bg / illustration / deco / artwork”等语义，不要误命名成 `banner-full.png` 这类整卡含文案资源。
- 同一页面的同类资源尽量统一前缀，例如：
  - `stage-training-banner-bg.png`
  - `stage-training-badge-gold.png`
  - `stage-training-card-pattern.png`
- 页面中默认使用本地导入地址，不再默认提供服务器图片地址。
- 只有用户明确要求走服务器资源时，才额外补充服务器路径方案。

## 输出要求

- 优先给出：目标页面路径、`CSS 可还原` 清单、`建议导图` 清单、去文案/整卡判断、已导出的本地路径、本地导入路径、缺失项。
- 少贴大段代码，重点说明判断结果和落库结果。
- 如与 `module-handUI` 一起使用，要明确哪些视觉点已通过 CSS 处理，哪些依赖导图资源。
- 若用户后续还要用 `module-handUI`，清单中的每张图都要能直接映射到页面里的一个视觉区块，避免只给文件名不给用途。

## 禁止事项

- 不要把可稳定用 CSS 实现的简单元素一律切图。
- 不要在没有素材的情况下伪造位图文件。
- 不要在已经具备 Figma 导出条件时，只给清单不导图。
- 不要在用户已经指定资源目录后，擅自改用别的目录。
- 不要把图片资源散落到页面目录或新建随意目录；未指定时统一放到项目默认图片目录，例如 `<project_asset_dir-src/assets/images/figma/>`。
- 不要把同一素材因循环渲染、多处摆放、重复组件实例或仅有轻微视觉差别而重复导出成多张高度相似的图片。
- 不要只导图不登记；只要本轮判断过，就同步更新 `<asset_registry_doc_path-docs/modules/Figma-images.md>`。
- 不要把带业务文案、按钮、标签的整卡一股脑导出成一张图，除非已经明确判断无法拆分。
- 不要根据模糊截图臆测品牌图形、插画细节或透明边界。

## 标准触发模板

适合直接给 AI 的请求模板：

`使用 figma-asset-triage-template，基于 [Figma 链接/节点/截图] 处理 [页面路径]，资源目录为 [目录名，可选]。请判断哪些元素继续用 CSS 还原、哪些元素应导图；图标优先复用项目现有图标体系或 <ui_library-Element UI>。若一个视觉区块同时包含插画和业务文案/按钮/标签，优先导出无文案底图或插画层，不默认整卡导出。若能从 Figma 导出，请实际导出到项目约定的本地目录，格式默认优先 png；多处图片只要肉眼看上去基本是同一张，就只导一张并复用，除非肉眼明显不同。若未指定资源目录，默认按 <project_asset_dir-src/assets/images/figma/> 和 <project_asset_import_base-@/assets/images/figma/> 处理。请把结果追加到 <asset_registry_doc_path-docs/modules/Figma-images.md>；若素材不足，请明确列出缺失项。`
